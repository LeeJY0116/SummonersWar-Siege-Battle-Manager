# 한글 인코딩 작업 규칙

이 프로젝트의 소스 파일은 UTF-8을 기준으로 관리한다.

## 기본 원칙

- UI 문구와 메시지는 `"\uXXXX"` 이스케이프가 아니라 한글을 직접 작성한다.
- `\uXXXX`는 외부 제약 때문에 꼭 필요한 경우에만 사용한다.
- 한글을 포함한 파일은 PowerShell의 `Get-Content | Set-Content`, `Out-File`, 리다이렉션 쓰기(`>`, `>>`)로 수정하지 않는다.
- 수동 수정은 `apply_patch`를 우선 사용한다.
- 대량 기계 변환은 UTF-8을 명시한 Node 스크립트처럼 바이트/인코딩이 명확한 방식만 사용한다.

## 수정 후 확인

한글이 포함된 파일을 수정한 뒤에는 아래 패턴을 검색한다.

```powershell
rg -n "�|占|紐|願|뚯|놁|덈|좉|ぉ|\?\?\?" <수정한 파일 또는 src 경로>
```

프론트 파일을 수정한 뒤에는 빌드를 실행한다.

```powershell
npm run build
```

## 깨진 한글 복구 원칙

- 이미 깨진 문자열은 추측해서 한꺼번에 복구하지 않는다.
- 화면 문맥, 기존 정상 문구, 사용자 확인을 기준으로 작은 단위로 복구한다.
- 기능 변경과 대량 문구 복구는 같은 커밋에 섞지 않는다.
