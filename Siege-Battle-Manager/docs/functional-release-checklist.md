# Functional Release Checklist

운영 배포 전후에 실제 사용자 흐름 기준으로 확인할 기능 점검 목록이다.

## 계정과 승인

- 로그인 실패 시 "아이디 또는 비밀번호가 올바르지 않습니다." 메시지가 표시된다.
- 승인 전 길드장 가입 요청 계정은 로그인할 수 없다.
- 승인 전 길드원 가입 요청 계정은 로그인할 수 없다.
- admin은 길드 개설 요청을 승인/거절할 수 있다.
- 길드장은 길드원 가입 요청을 승인/거절할 수 있다.
- 거절된 가입 요청은 같은 ID, 닉네임, 이메일 재신청을 막지 않는다.

## 내 정보와 길드 상태

- 로그인 후 내 정보 탭에서 ID, 이메일, 닉네임, 소속 길드를 확인할 수 있다.
- 길드 탈퇴 후에는 가입/개설 요청 화면만 접근 가능하다.
- 길드 탈퇴 후 다른 길드 가입 요청을 보낼 수 있다.
- 가입 요청 중에는 다른 길드 가입/개설 요청을 중복으로 보낼 수 없다.

## 회원 관리

- 길드장은 회원 관리 탭을 볼 수 있다.
- 부길드장은 회원 관리 탭을 볼 수 있다.
- 일반 길드원은 회원 관리 탭을 볼 수 없다.
- 길드장은 길드원 등급 변경, 추방, 재가입 차단 해제를 처리할 수 있다.
- 길드장 양도 후 우측 상단 길드장 표시가 새 길드장으로 바뀐다.
- 추방된 계정은 재가입 불가 목록에 표시되고, 해제 전에는 재가입할 수 없다.

## 인벤토리

- 일반 길드원은 본인 인벤토리만 조회/수정할 수 있다.
- 길드장과 부길드장은 같은 길드원의 인벤토리를 조회/수정할 수 있다.
- 다른 길드 인벤토리는 직접 URL/API를 알아도 접근할 수 없다.
- 몬스터 수량은 0~10 범위를 벗어나지 않는다.
- 방덱에 사용 중인 몬스터 수량보다 낮게 줄일 수 없다.

## 방덱

- 일반 길드원은 본인 방덱만 생성/삭제할 수 있다.
- 길드장과 부길드장은 같은 길드원의 방덱을 생성/삭제할 수 있다.
- 방덱 소유 길드원을 변경하면 선택된 몬스터 슬롯이 초기화된다.
- 리더가 다르면 같은 몬스터 3종이어도 다른 방덱으로 취급된다.
- 4성 토글 활성화 시 5성 미포함 방덱만 표시된다.
- 길드 탈퇴자의 방덱은 생성 가능 길드원 수에 포함되지 않는다.

## 길드 방덱

- 길드장은 더미 방덱을 생성/삭제할 수 있다.
- 길드 방덱 목록에서 리더 효과, 4성 토글, 검색/필터가 동작한다.
- 생성 가능 길드원 수는 현재 승인된 길드원 기준으로 계산된다.
- 상세 보기에서 소유 가능 길드원 목록이 현재 길드 상태와 맞는다.

## 전투 연구

- 전투 연구 글은 30초에 한 번 작성할 수 있다.
- 댓글은 30초에 한 번 작성할 수 있다.
- 글 작성 실패 시 입력 중인 제목/본문이 유지된다.
- 댓글 작성 실패 시 입력 중인 댓글 내용이 유지된다.
- 글 목록은 10개 단위로 페이지네이션된다.
- 댓글은 10개 단위로 페이지네이션된다.
- 작성자와 길드장만 글/댓글을 삭제할 수 있다.
- 다른 길드 전투 연구 글/댓글은 직접 ID를 알아도 조회/수정/삭제할 수 없다.

## 몬스터 관리

- admin 계정에서만 몬스터 동기화 버튼이 표시된다.
- admin 계정에서만 도감 정보 적용 버튼이 표시된다.
- 몬스터 동기화와 도감 정보 적용은 동시에 실행되지 않는다.
- 작업 중에는 진행 상태가 표시된다.
- 서버 재시작 후 진행 중이던 작업은 실패 상태로 표시되고 다시 실행할 수 있다.
- 숨김 처리된 몬스터는 사용자 선택 목록에 표시되지 않는다.
- 리더 효과는 전체, 길드 전투, 속성 리더만 필터에 반영된다.

## 관리자 페이지

- admin은 회원 관리, 길드 관리, 몬스터 관리를 볼 수 있다.
- 일반 사용자는 `/api/admin/**`에 접근할 수 없다.
- 회원 관리에서 ID, 닉네임, 길드, 이메일, IP 검색이 동작한다.
- 길드 관리에는 현재 존재하는 길드만 표시된다.
- 길드 해체 후 해당 길드는 가입 가능 길드/개설 중복 검사에 남지 않는다.
- 강제 계정 삭제 후 같은 ID와 닉네임으로 재가입할 수 있다.

## 운영 Smoke Test

아래 스크립트가 통과해야 한다.

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\operational-smoke-test.ps1
```

스크립트가 확인하는 항목:

- 프론트 URL 200 응답
- 백엔드 미로그인 401 JSON 응답
- admin API 미로그인 401 JSON 응답
- `/api/monsters` 200 응답
- 운영 CORS 허용 Origin
