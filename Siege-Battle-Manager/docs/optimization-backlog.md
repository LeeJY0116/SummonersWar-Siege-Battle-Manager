# Optimization Backlog

운영 중 체감 속도나 비용에 영향을 줄 수 있는 최적화 후보를 기록한다.

## 몬스터 목록 캐싱

### 배경

로그인 후 내 정보, 길드 정보, 길드원 목록, 인벤토리, 몬스터 목록을 연달아 조회하면서 화면 표시까지 1~2초 정도 걸릴 수 있다.

몬스터 목록은 자주 바뀌지 않고 응답 크기도 비교적 크기 때문에 캐싱 후보로 둔다.

### 후보 방향

- 프론트에서 `/api/monsters` 응답을 메모리 또는 localStorage에 캐싱한다.
- 백엔드에서 `ETag` 또는 `Last-Modified` 기반 조건부 요청을 검토한다.
- 도감 정보 적용 또는 Swarfarm 동기화 완료 시 캐시 무효화 기준을 둔다.

### 우선순위

현재는 기능 안정화와 배포 안정화가 우선이다. 사용자 수가 늘거나 초기 화면 지연이 반복적으로 체감되면 진행한다.
