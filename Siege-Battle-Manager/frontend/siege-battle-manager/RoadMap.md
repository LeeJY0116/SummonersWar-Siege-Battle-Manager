- 길드원별 몬스터 관리
- 방어 덱 구성
- 방덱 가능 인원 계산
- 공덱 전략 공유

# ✅ 1차 개발 - 프론트엔드 기본 구조 구축 (완료)

## 목표

점령전 방덱을 빠르게 구성하고 조회할 수 있는 UI 제작

## 구현 내용

### 몬스터 관리
- 몬스터 등록
- 기본 몬스터 데이터 JSON 구성
- 속성별 분류
- 몬스터 검색
- 닉네임 검색

---

### 방덱 시스템

- 3마리 방덱 생성
- 리더 몬스터 개념 적용
- 리더 효과 표시
- 리더 효과 필터링
- 방덱 저장/삭제

---

### UI 기능

- 속성 아이콘 필터
- 우측 고정 필터 UI
- 이미지 기반 선택 UI
- 몬스터 포함 필터
- 최대 3마리 동시 필터
- 4성 방덱 필터

---

## 기술 스택

- React
- Vite
- Tailwind CSS

# ✅ 2차 개발 - 백엔드 및 인증 시스템 구축 (완료)

## 목표

프론트 단독 구조에서 실제 서비스 구조로 전환

---

## 구현 내용

### 사용자 인증

- 회원가입 API
- 로그인 API
- JWT 발급
- JWT 검증
- Spring Security 적용
- JwtAuthenticationFilter 구현

---

### 사용자 정보 관리

- /api/users/me 구현
- 인증 사용자 조회
- Authorization Header 자동 처리

---

### 공통 응답 구조

ApiResponse<T> 구조 도입

```json id="k58m9m"
{
  "success": true,
  "data": {},
  "message": "성공"
}
```
## 기술 스택

- Java 17
- Spring Boot 3
- Spring Security
- JWT
- JPA
- H2

# ✅ 3차 개발 - 길드 시스템 구축 (완료)

## 목표

길드 단위 관리 기능 구현

---

## 구현 내용

### 길드 기능

- 길드 생성
- 내 길드 조회
- 길드원 조회

---

### 권한 시스템

| 권한         | 설명       |
| ---------- | -------- |
| MASTER     | 길드 전체 관리 |
| SUB_MASTER | 운영 권한    |
| MEMBER     | 일반 길드원   |


---

### 가상 길드원 구조 설계

실제 계정이 없어도:

- 인벤토리 등록 가능
- 방덱 구성 가능
하도록 설계


- 실제 웹사이트 운영 로직
