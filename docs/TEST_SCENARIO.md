# 📌 테스트 목적

포트폴리오 확인자 또는 개발자가 로컬 환경에서 핵심 기능 흐름을 빠르게 검증할 수 있도록 정리한 시나리오입니다.

---

# 🧪 테스트 계정

로컬 H2 DB에 아래 계정이 있으면 이 계정으로 테스트합니다.

```text
loginId: test
email: test@test.com
password: test
nickname: test
guild: test
```

계정이 없다면 길드장 가입 화면에서 같은 정보와 `test` 길드 이름으로 가입 신청합니다.
단, 길드장 가입 신청은 관리자 승인 전까지 로그인할 수 없습니다.

# ✅ 사전 준비

## 1) 백엔드 실행

```bash
cd Siege-Battle-Manager/backend/siege-backend
./gradlew bootRun
```

기본 API 주소:

```text
http://localhost:8080
```

## 2) 프론트엔드 실행

```bash
cd Siege-Battle-Manager/frontend/siege-battle-manager
npm install
npm run dev
```

기본 화면 주소:

```text
http://localhost:5173
```

# ✅ 기본 검증 흐름

## 1) 인증

1. `test / test`로 로그인합니다.
2. 계정이 없으면 길드장 가입으로 `test` 길드 가입 신청을 생성합니다.
3. 관리자 계정으로 가입 신청을 승인한 뒤 다시 로그인합니다.
4. 로그인 전 승인 대기 계정은 승인 대기 메시지가 표시되는지 확인합니다.
5. 로그인 후 상단에 길드 정보가 표시되는지 확인합니다.

## 2) 가입 / 승인 흐름

### 길드장 가입

1. 시작 화면에서 `길드장 가입`을 선택합니다.
2. 아이디, 비밀번호, 비밀번호 확인, 길드 이름, 인게임 닉네임, 이메일을 입력합니다.
3. 가입 신청 완료 후 로그인하면 `관리자 승인 후 로그인할 수 있습니다.` 메시지가 표시되는지 확인합니다.
4. 관리자 계정으로 로그인해 길드장 가입 신청을 승인합니다.
5. 승인 후 해당 계정으로 로그인하면 길드가 생성되고 본인이 `길드장`으로 표시되는지 확인합니다.

### 길드원 가입

1. 시작 화면에서 `길드원 가입`을 선택합니다.
2. 이미 승인된 길드 이름을 입력해 가입 신청합니다.
3. 승인 전 로그인하면 `길드장 승인 후 로그인할 수 있습니다.` 메시지가 표시되는지 확인합니다.
4. 길드장 또는 부길드장 계정으로 회원 관리 탭에서 가입 신청을 승인합니다.
5. 승인 후 길드원 계정으로 로그인하면 길드 정보가 표시되는지 확인합니다.

### 기존 계정 재가입

1. 길드가 없는 기존 계정으로 로그인합니다.
2. 길드 가입 요청 전용 화면만 표시되는지 확인합니다.
3. 길드 이름을 입력해 가입 요청을 보냅니다.
4. 요청 대기 중에는 다른 길드 가입 요청을 보낼 수 없고, 철회 버튼이 표시되는지 확인합니다.
5. 요청을 철회하면 다시 가입 요청을 보낼 수 있는지 확인합니다.
6. 길드장 또는 부길드장 승인 후 일반 기능에 접근 가능한지 확인합니다.

### 거절 / 차단

1. 길드장 가입 신청을 거절한 뒤 같은 아이디로 다시 신청할 수 있는지 확인합니다.
2. 길드원을 추방하면 회원 관리 탭의 재가입 불가 목록에 추가되는지 확인합니다.
3. 재가입 불가 목록에 있는 계정이 같은 길드에 재가입 요청하면 차단 메시지가 표시되는지 확인합니다.
4. 재가입 차단을 해제하면 다시 가입 요청을 보낼 수 있는지 확인합니다.

## 3) 길드

1. 상단에 `길드: test`, `마스터: test`가 표시되는지 확인합니다.
2. 길드원 목록에서 본인 계정이 표시되는지 확인합니다.
3. 회원 관리 탭에서 길드원이 등급순, 최근 접속일순으로 정렬되는지 확인합니다.
4. 회원 관리 탭의 인원 표시가 `현재/35` 형식으로 표시되는지 확인합니다.
5. 길드장 양도 후 상단의 `마스터` 표시가 새 길드장 닉네임으로 갱신되는지 확인합니다.
6. 더미 계정을 생성하면 길드 인원 수에 포함되는지 확인합니다.
7. 더미 계정을 삭제하면 길드 인원 수가 감소하는지 확인합니다.

## 4) 몬스터 데이터

1. 관리자 기능에서 `Sync Swarfarm`을 실행합니다.
2. 이어서 `Apply Names`를 실행합니다.
3. 도감 또는 몬스터 선택 UI에서 한글명과 별칭 검색이 동작하는지 확인합니다.
4. 신규 몬스터는 Swarfarm 동기화로 DB에 추가되지만, 사용자 노출은 `monster-localization.json`의 `koreanName`, `enabled`, `awakeningLevel` 기준을 따르는지 확인합니다.

관리 파일:

```text
Siege-Battle-Manager/backend/siege-backend/src/main/resources/data/monster-localization.json
```

## 5) 길드원 인벤토리

1. 길드 탭에서 길드원 인벤토리를 입력합니다.
2. 몬스터 이름, 한글명, 별칭 검색이 동작하는지 확인합니다.
3. 저장 후 새로고침해도 수량이 유지되는지 확인합니다.
4. 몬스터 수량을 10보다 크게 입력하면 10으로 보정되는지 확인합니다.
5. 몬스터 수량을 음수로 입력하면 0으로 보정되는지 확인합니다.
6. 이미 방덱에 사용 중인 수량보다 낮게 줄일 수 없는지 확인합니다.

## 6) 방덱 생성

1. 길드원 방덱 생성 화면에서 몬스터 3마리를 선택합니다.
2. 첫 번째 몬스터가 리더로 표시되는지 확인합니다.
3. 방덱 생성 후 인벤토리 수량이 차감되는지 확인합니다.
4. 방덱 삭제 후 수량이 복구되는지 확인합니다.
5. 리더 효과 필터와 포함 몬스터 검색이 동작하는지 확인합니다.
6. 방덱 소유 길드원을 변경하면 선택된 몬스터 슬롯이 초기화되는지 확인합니다.

## 7) 길드 방덱

1. 길드 방덱을 생성합니다.
2. 상세 보기에서 생성 가능 길드원 목록이 표시되는지 확인합니다.
3. 길드원이 탈퇴한 뒤 생성 가능 길드원 수에서 제외되는지 확인합니다.
4. 생성 가능 길드원 목록에 `LEFT` 상태 멤버가 표시되지 않는지 확인합니다.

## 8) 전투 연구

1. 전투 연구 게시글을 작성합니다.
2. 연구 대상 방덱 몬스터가 한글명으로 표시되는지 확인합니다.
3. 댓글을 작성하고 수정/삭제가 가능한지 확인합니다.
4. 게시글 삭제 권한이 작성자/마스터/부마스터 기준으로 동작하는지 확인합니다.
5. 제목 100자, 본문 3000자, 댓글 1000자 제한이 동작하는지 확인합니다.
6. 게시글 또는 댓글을 연속 작성하면 30초 제한 메시지가 표시되는지 확인합니다.
7. 작성 실패 후 입력 중인 내용이 유지되는지 확인합니다.
8. 게시글과 댓글이 10개 단위로 페이지네이션되는지 확인합니다.
9. 리더 효과, 포함 몬스터, 4성 방덱 필터가 전체 게시글 기준으로 동작하는지 확인합니다.
