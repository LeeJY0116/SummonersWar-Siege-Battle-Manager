# 2026-07-13 작업 기록

## 요약

오늘은 배포 전 실제 사용 흐름에 맞춰 길드 운영, 전투 연구 제한, 방덱 관리 UX, 사용자 화면 UI 톤을 정리했다.

## 완료한 작업

### 길드 운영

- 길드 내 더미 계정 생성/삭제 기능 추가
- 더미 계정도 길드 인원 제한 35명에 포함
- 회원 관리 탭에서 실제 계정과 더미 계정 수를 `현재/35` 형식으로 표시
- 길드원 추방 후 재가입 불가 목록 관리
- 길드장 양도 후 상단 길드 마스터 정보 갱신
- 회원 관리 탭의 길드 멤버 목록에서 최근 접속일 표시
- 길드 멤버 정렬 기준을 등급, 최근 접속일 순으로 정리

### 인벤토리와 방덱

- 몬스터별 인벤토리 수량을 최대 10마리로 제한
- 직접 입력 시 10 초과는 10, 음수는 0으로 보정
- 방덱 소유 길드원을 변경하면 기존 선택 몬스터 슬롯을 초기화
- 길드 방덱의 생성 가능 길드원 계산에서 탈퇴한 `LEFT` 멤버 제외

### 전투 연구

- 게시글 제목 100자, 본문 3000자, 댓글 1000자 제한 추가
- 게시글/댓글 작성은 30초에 1회로 제한
- 작성 실패 시 작성 중이던 글/댓글 내용 유지
- 전투 연구 게시글과 댓글을 10개 단위로 백엔드 페이지네이션
- 전투 연구 목록 검색/필터를 백엔드 요청 파라미터 기반으로 변경
  - 리더 효과 필터
  - 포함 몬스터 필터
  - 4성 방덱 필터

### 사용자 화면 UI

- 사이트 표시명을 `SW 점령전`으로 변경
- 사용자 화면을 갈색/금색 기반 게임 UI 톤으로 정리
- 좌측 탭 설명 카드 추가
- 길드, 인벤토리, 전투 연구, 방덱, 길드 방덱, 내 정보, 회원 관리 탭의 외곽 톤 통일
- 회원 관리 탭의 한글 깨짐 문구 복구

## 트러블슈팅

### 길드 방덱 생성 가능 길드원에 탈퇴자가 포함되는 문제

길드 방덱의 생성 가능 인원을 계산할 때 `GuildMember` 전체 row를 기준으로 계산해 탈퇴한 `LEFT` 멤버도 포함되었다.  
`OwnerlessDefenseDeckService`에서 `GuildMemberStatus.APPROVED` 상태인 현재 길드원만 대상으로 필터링해 해결했다.

### 방덱 생성 중 소유 길드원 변경 시 몬스터가 섞이는 문제

방덱 생성 화면에서 소유 길드원을 바꿔도 기존 선택 몬스터가 남아 있었다.  
`ownerMemberId` 변경 시 선택 몬스터 슬롯, 활성 슬롯, 검색어를 초기화하도록 수정했다.

### 전투 연구 검색/필터가 현재 페이지 안에서만 동작할 수 있는 문제

기존 구조에서는 프론트가 받아온 현재 페이지 안에서만 필터링될 수 있었다.  
검색/필터 조건을 백엔드 요청 파라미터로 전달하고, 백엔드에서 전체 게시글 기준으로 필터링 후 페이지네이션하도록 변경했다.

### 사용자 화면 한글 깨짐과 UI 톤 불일치

일부 사용자 화면 문구가 깨져 보이고 흰색 카드와 갈색 카드가 섞여 있었다.  
사용자에게 노출되는 주요 화면부터 정상 한글과 고정형 게임 UI 톤으로 정리했다.
