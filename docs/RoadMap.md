# ✅ 8차 개발 - 회원 승인 및 운영 이력 관리 (완료)

## 구현 내용

### 가입 승인 구조
- 길드장 가입 신청은 관리자 승인 전까지 실제 길드와 계정을 생성하지 않도록 변경
- 길드원 가입 신청은 길드장/부길드장 승인 전까지 로그인 불가
- 거절된 가입 신청은 실제 계정을 만들지 않아 같은 아이디로 재신청 가능
- 승인 대기 상태 로그인 시 승인 대기 메시지 표시

### 기존 계정 재가입 흐름
- 길드가 없는 기존 계정은 길드 가입 요청 전용 화면만 접근
- 가입 요청 중에는 다른 길드로 중복 신청 불가
- 사용자가 대기 중인 가입 요청을 철회 가능
- 승인 시 새 `GuildMember` row를 생성해 이전 길드 이력과 현재 소속을 분리

### 회원 관리
- 길드장/부길드장 전용 회원 관리 탭 추가
- 길드원 가입 승인/거절
- 길드원 등급 변경
- 길드원 추방
- 추방된 길드원의 재가입 차단 목록 관리

### 관리자 기능
- 관리자 전용 길드 목록 및 길드원 현황 조회
- 현재 소속 길드 표시
- 탈퇴/재가입 이력 조회
- 닉네임 변경 요청 승인/거절
- 닉네임 변경 이력 조회

# 📚 개발 중 경험한 주요 문제

## 백엔드 완료
- JWT 인증 문제
- Authorization Header 누락 문제
- N+1 문제
- Fetch Join 최적화
- H2 Unique Constraint 문제
- JSON 기반 몬스터 데이터 처리 문제
- Monster ID / Monster Code 혼동 문제
- Swarfarm 동기화와 로컬 관리 데이터 병합 문제
- 한글 인코딩 깨짐 방지 규칙 정리
- 가입 승인 구조와 실제 계정 생성 시점 분리 문제
- 탈퇴 이력 row와 현재 멤버십 조회 충돌 문제
- 재가입 차단 목록 응답 형태 불일치 문제
- `IllegalStateException` 미처리로 500 응답이 발생한 문제

※ 자세한 내용은 docs/TROUBLE_SHOOTING.md 참고
