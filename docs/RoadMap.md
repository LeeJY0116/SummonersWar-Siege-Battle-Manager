# SW 점령전 로드맵 및 개발 기록

> Last Updated: 2026-07-19

## 프로젝트 목표

서머너즈워 점령전 운영에 필요한 길드원 인벤토리, 공·방덱 조합, 전투 연구와 실제 점령전 로그 통계를 길드 단위로 관리한다.

핵심 원칙:

- 몬스터 식별은 `monsterCode`를 우선한다.
- 사이트 길드는 서로 완전히 격리한다.
- 점령전 통계의 닉네임과 길드 정보는 사이트 회원 정보가 아니라 업로드 JSON을 기준으로 한다.
- 운영 비밀값과 실제 로그는 Private 저장소와 배포 환경변수에서만 관리한다.

---

## 1차 개발: 인증과 길드 운영 (완료)

- ID 기반 회원가입과 JWT 로그인
- 길드장 가입의 관리자 승인과 길드원 가입의 길드 승인
- 승인 전 로그인 차단, 가입 요청 철회, 추방 및 재가입 차단
- 닉네임 변경 요청과 관리자 이력 조회
- 길드장·부길드장·길드원 권한 분리
- 길드 정원 35명과 더미 계정 관리
- 길드장 양도, 길드 탈퇴 및 길드 해체

---

## 2차 개발: 몬스터와 인벤토리 (완료)

- Swarfarm 몬스터 데이터 비동기 동기화
- localization JSON 기반 한글 이름, 별칭, 노출 여부 관리
- 각성 단계와 콜라보·오리지널 대응 관계 관리
- 전체·길드 전투·속성 리더 효과만 적용
- 몬스터 목록 메모리 및 브라우저 캐싱
- 길드원별 몬스터 수량 0~10 제한
- 성급·속성 필터와 이름·별칭 검색
- 길드 탈퇴 시 인벤토리 초기화

---

## 3차 개발: 방덱 관리 (완료)

- 길드원 인벤토리 기반 방덱 생성·삭제
- 길드 공용 방덱 등록과 생성 가능 길드원 계산
- 리더 슬롯과 나머지 두 슬롯을 구분한 중복 판정
- 리더 효과, 4성 방덱, 포함 몬스터 검색·필터
- 동일 방덱의 보유 수와 소유 길드원 표시
- 권한별 생성·수정·삭제 범위 검증

---

## 4차 개발: 회원·관리자 운영 도구 (완료)

- 회원·길드·몬스터 관리 탭과 관리자 편집 모드
- 회원 검색, 닉네임·길드 이력, 최근 접속일·IP 표시
- 계정 강제 삭제, 길드 등급 변경 및 길드 해체
- 관리자 전용 길드 읽기 모드와 대시보드 요약
- 몬스터 동기화와 도감 정보 적용의 중복 실행 방지 및 상태 조회

---

## 5차 개발: 점령전 로그 통계 (완료, 운영 검증 중)

### 로그 수집과 관리

- SWEX JSON 로그 업로드
- 파일·전투 단위 중복 방지
- 우리 길드와 상대 길드, 공격·방어 관점 및 승패 정규화
- 업로드 이력 조회와 개별 삭제
- 길드 해체 시 해당 길드 로그 삭제

### 통계

- 공덱·방덱 조합별 전적과 승률
- 공격자, 방어자, 소유자 및 길드별 상세 전적
- 전체·우리 길드·상대 길드 방덱 구분
- 4성 조합 필터와 포함 몬스터 검색
- 방덱 목록 50개 단위 페이징
- 콜라보·오리지널 몬스터 통합 집계
- 유효한 리더가 없는 조합의 전체 정렬 정규화
- Caffeine 캐시와 PostgreSQL/H2 조회 인덱스

통계 데이터는 사이트 회원 도메인과 결합하지 않는다. 사이트 길드는 JSON 통계 데이터에 접근할 수 있는 보안 경계로만 사용한다.

---

## 6차 개발: 전투 연구 V2 (완료, 운영 검증 중)

- 방덱과 공덱 조합 독립 관리
- 방덱·공덱 N:N 매치업 연결
- 매치업별 공략 작성·삭제
- 조합 검색 팝오버, 별칭 검색, 4성 필터
- 조합 목록과 매치업 10개 단위 페이징
- 점령전 로그 기반 실제 사용 전적·승률 자동 연동
- 사용 로그 10개 단위 조회
- 조합 삭제와 매치업 연결 해제 분리
- 반대편 조합 보존 및 길드 간 접근 차단 테스트

레거시 게시글·댓글 API는 남아 있지만 사용자 화면은 V2를 사용한다. 안정화 후 레거시 데이터와 API 제거 여부를 결정한다.

---

## 7차 개발: 배포와 운영 기반 (완료)

```text
Cloudflare Pages -> Render Singapore -> Neon PostgreSQL Singapore
```

- 개발 환경: H2와 `dev` 프로필
- 운영 환경: PostgreSQL과 `prod` 프로필
- Flyway 운영 스키마 마이그레이션
- Render Docker 배포와 Cloudflare Pages 프론트 배포
- CORS, 보안 헤더, Rate Limit, 관리자 API 권한 적용
- Public 포트폴리오 저장소와 Private 실서비스 저장소 분리
- 운영 smoke test, 기능 체크리스트, 백업·복구 및 베타 초기화 절차 작성

---

## 현재 우선순위

### 1. 베타 테스트 안정화

- 실제 길드 데이터로 통계 조합 정규화 검증
- 전투 연구 V2 등록·연결·삭제 전체 흐름 점검
- 모바일 UI와 긴 목록의 사용성 확인
- 운영 오류와 느린 캐시 미스 쿼리 기록

### 2. 베타 데이터 초기화와 정식 운영 준비

- Neon 백업 생성
- 사용자·길드·전투 연구·점령전 로그 초기화
- admin과 몬스터 기준 데이터 보존 확인
- 운영 환경 smoke test 재실행
- 사용자 안내와 개인정보 처리 범위 정리

### 3. 이메일 기능

- SMTP 서비스 선정과 발신 도메인 인증
- 가입 승인·거절 안내 메일
- 비밀번호 재설정 토큰과 만료 처리
- 메일 발송 실패 로그와 재시도 정책

---

## 장기 후보

실제 필요가 확인된 뒤 진행한다.

- 통계 원본 로그가 커졌을 때 매치·일별 집계 테이블 도입
- 커스텀 도메인과 Cloudflare WAF 추가 설정
- 공식 API를 제공하는 플랫폼의 길드 전용 챗봇 연동
- 채팅 명령으로 공덱·방덱·매치업·공략 등록
- 다중 인스턴스 운영이 필요할 때 Redis 기반 공유 캐시
