🏰 Siege-Battle-Manager

길드 점령전 방덱/공덱 관리 & 길드 운영 플랫폼

📌 소개

Siege-Battle-Manager는 점령전(길드 전쟁) 컨텐츠를 효율적으로 운영하기 위한
길드 전용 방덱 관리·전투 연구 시스템이다.

점령전에서 중요한 것은 길드 전체의 방덱 구성 능력이다.
그러나 실제 게임에서는:

길드원마다 보유 몬스터가 다르고

방덱 템플릿이 몇 세트나 가능한지 확인하기 어렵고

길드원 전부가 사이트에 가입하는 것도 쉽지 않다

이 문제를 해결하기 위해 생성된 프로젝트가 Siege-Battle-Manager이다.

🎯 핵심 목적

길드 전체 방덱 현황을 시각화

각 길드원이 구성 가능한 방덱을 정확히 계산

같은 방덱 템플릿이 몇 세트 가능한지 자동 분석

전투 연구(공덱 연구/전략 아카이브) 저장

길드 마스터/부마스터가 길드를 효율적으로 관리할 수 있게 함

🗂️ 구성 요소 요약

이 프로젝트는 크게 3개의 큰 기능 그룹으로 나뉜다:

길드원 시스템

방덱 관리(길드 방덱 탭)

전투 연구 탭(Battle Research)

각 기능은 완전한 서비스처럼 독립적으로 동작하면서도 서로 유기적으로 연결된다.

1️⃣ 길드 시스템
👑 길드 역할 구조
역할	권한
마스터	전체 권한, 부마스터 임명/해임, 길드원 관리, 방덱 구성/삭제, 인벤토리 수정, 길드원 추방, 길드 해체 가능
부마스터(최대 5명)	마스터와 거의 동일한 권한 (길드 해체만 불가)
길드원	자신의 인벤토리 수정, 자신의 방덱 구성/삭제, 연구탭 작성 가능
✔ 길드 최대 인원: 35명
2️⃣ 길드원 인벤토리 (보유 몬스터 관리)

각 길드원은 다음 정보를 가진다:

보유 몬스터 목록

각 몬스터별 보유 개수

입력/수정 권한

본인 → 자신의 인벤토리 수정 가능

마스터/부마스터 → 모든 길드원 인벤토리 수정 가능

3️⃣ 임의 길드원(가짜 유저) 기능

길드 전체가 사이트에 회원가입하기 어려울 수 있기 때문에,
마스터/부마스터는 임의 길드원을 추가할 수 있다.

이 임의 길드원은:

로그인 계정이 아니며

길드 마스터/부마스터가 이름·인벤토리·방덱을 설정 가능

언제든지 삭제 가능

삭제해도 전투 연구 탭에 작성한 글은 유지됨

4️⃣ 길드 방덱 관리 (Defense Deck Tab)

방덱은 3마리 몬스터로 구성되며 첫 번째 몬스터가 리더이다.

✔ 방덱 구성 규칙

구성한 방덱에 사용된 몬스터는 보유 수량에서 -1

방덱 삭제 시 +1 복구

가능한 방덱 개수를 정밀하게 계산할 수 있음

✔ 권한
누가?	자신의 방덱	    다른 길드원 방덱
길드원	생성/삭제 가능	        X
부마스터	O	                O
마스터	    O	                O
5️⃣ 주인 없는 방덱 (Ownerless Defense Deck)

운영진(마스터 + 부마스터)은 길드원의 소유가 아닌
**방덱 템플릿(주인 없는 방덱)**을 등록할 수 있다.

목적

예:
암 네오스톤 에이전트 + 암 마카롱 친위대 + 풍 젠이츠 방덱을
길드 전체에서 몇 명이 이걸 만들 수 있는지 알고 싶을 때 사용.

기능

템플릿 생성

시스템이 자동으로 “이 방덱이 가능한 길드원 리스트” 계산

방덱 유지/폐기 판단 기준으로 활용

6️⃣ 방덱 필터링 시스템

방덱 목록은 아래 조건으로 필터링할 수 있다.

🔍 1) 몬스터 필터

특정 몬스터를 클릭하면:

그 몬스터가 포함된 방덱만 표시

정렬 우선순위:

해당 몬스터가 리더인 방덱

서브로 포함된 방덱

🔍 2) 리더 효과 필터

예: “공격속도”
→ 리더 효과가 공격속도인 방덱만 표시

🔍 3) 중첩 가능

몬스터 + 리더효과 필터 동시에 적용 가능

길드 탭에서도 동일한 필터 사용

7️⃣ 전투 연구 탭 (Battle Research)

전투 연구 탭은 길드원들이
방덱 연구, 공덱 조합, 운용법, 전략 코멘트를 공유하는 공간이다.

특징

인벤토리와 완전히 독립된 기능

모든 몬스터 중 자유롭게 방덱 설정 가능

실전 경험 및 팁을 아카이브용으로 저장됨

📄 7-1. 게시글 (방덱 연구 글)
게시글 내용

방덱(방어 세 마리)

제목

작성자 / 작성일

댓글 수


삭제 시

게시글과 그 아래의 모든 댓글(공덱 조합)이 삭제됨

💬 7-2. 댓글 (공덱 조합 + 코멘트)

댓글 내용:

공덱 조합(0~3마리)

전략/운용 코멘트

❗ 7-3. 길드원 탈퇴와 전투 연구 탭의 독립성
가장 중요한 규칙

길드원 삭제/탈퇴는 전투 연구 탭에 절대 영향을 미치지 않는다.

길드원이 탈퇴해도:

그 사람이 작성한 게시글은 유지

댓글도 유지

글쓴이 정보도 유지

이는 전투 연구 탭이 지식/데이터 아카이브이기 때문이다.

9️⃣ 기술 스택

Frontend: React + Vite + Tailwind

Backend: Spring Boot 3.x, Java 17, Gradle

DB: H2 (개발), 향후 MySQL/PostgreSQL 가능

Auth: JWT 기반 인증/인가

ORM: Spring Data JPA (+ Hibernate)

Design: Guild-based permission system

🔟 프로젝트의 철학

이 프로젝트는 단순히 방덱을 저장하는 도구가 아니다.

길드 전체 방덱 전력을 계산하고

모든 길드원이 같은 목표를 향해 효율적으로 운영될 수 있게 하며

전략 연구 자료를 축적하는 데이터베이스 역할을 한다

즉, 점령전 운영을 시스템화한 길드 매니지먼트 플랫폼이다.
